得到一个Json节点

知道路径

  • 如果知道路径，就是XPATH那种，可以直接得到。

     val lat = (json \ "location" \ "lat").get
// returns JsNumber(51.235685)


val bigwig = (json \ "residents" \ 1).get
// returns {"name":"Bigwig","age":6,"role":"Owsla"}

  • 上面是官网上面的两个例子，返回的类型是一个JSObject

  • 如果路径不对，找不到，是会报错的，可以使用validate函数来做校验

    val bogusResult = (json \ "bogus").validate[String]
// JsError

  • 如果不使用get函数，如下，是不会报错的

    val maybeName  = (json \  "user" \ name)
只知道键值

  • 本来以为这种不支持的，没想到也是有办法，赞一个

    val names = json \\ "name"
// returns Seq(JsString("Watership Down"), JsString("Fiver"), JsString("Bigwig"))
通过index

  • 也可以通过index来得到节点值，

    val name = json("name")
// returns JsString("Watership Down")
#这个例子和上面的json \\ "name"的结果相比，就返回了第一个值。这点值得留意

val bigwig = json("residents")(1)
// returns {"name":"Bigwig","age":6,"role":"Owsla"}

Boolean

  • 工作中的一个bug，某个标志，在框架代码中定义的时候是java写的；
  • 调用的工程是scala的，所以直接使用了scala的Boolean。
  • 两个并不兼容。相当于框架中定义的函数并没有被调用，由于这个标志位并不是项目的测试重点，所以就漏到客户那边，才发现这个标志位在GUI上面没法被设置。因为框架的背景代码是java定义的，而GUI渲染时候是根据annotation去渲染的。就没成功。
  • 好吧，这个框架我还不够细致的了解啊。需要学习的细节太多了。
